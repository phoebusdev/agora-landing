Project Specification

Overview

Create a web-based dashboard application that provides a clean, professional interface for users to view and manage their data. The dashboard must work seamlessly on mobile phones, tablets, and desktop computers. Authentication is required before users can access any dashboard features.

Technical Requirements

  • Use modern web technologies (React, Vue, or vanilla JavaScript acceptable)
  • Responsive design that adapts to screen sizes from 320px to 2560px wide
  • Secure authentication with password hashing (bcrypt or similar)
  • Session management with secure tokens
  • Clean, professional UI following modern design principles
  • No external UI frameworks required (Tailwind CSS or custom CSS both acceptable)

Core Features

  • User Registration: New users can create an account with email and password
  • User Login: Existing users can sign in with their credentials
  • Dashboard View: Authenticated users see a personalized dashboard with sample data
  • Data Display: Dashboard shows at least 3 different types of data visualizations (tables, charts, or cards)
  • Navigation: Clear navigation between dashboard sections
  • Logout: Users can securely end their session

Deliverables

  • Live demo URL (deployed on Vercel, Netlify, or similar)
  • Test credentials for demo access
  • Brief technical explanation (500 words max)
  • Note any deployment instructions if demo requires specific setup

Acceptance Criteria

Your solution will be evaluated against these specific criteria. All must pass for acceptance.

1. User can create an account with email and password

Registration form accepts email and password, validates format, stores securely, and confirms account creation.

2. User can sign in with valid credentials

Login form accepts credentials, validates against stored data, creates session, and redirects to dashboard on success.

3. Dashboard displays correctly on mobile (375px width)

All UI elements are visible, readable, and functional when browser is resized to 375px width. No horizontal scrolling. Navigation works on touch devices.

4. Dashboard displays correctly on desktop (1920px width)

Layout takes advantage of wider screen, proper spacing and sizing, no overly stretched elements.

5. Dashboard shows at least 3 different data visualizations

Examples: data table, bar chart, stat cards, line graph. Sample data is acceptable. Visualizations must be clearly labeled and functional.

6. Unauthenticated users cannot access dashboard

Direct URL access to dashboard routes redirects to login. Session expiry is handled. No protected data visible without authentication.

7. User can log out and session is cleared

Logout button/link is visible in dashboard. Clicking it ends session and returns to login page. Cannot access dashboard after logout without signing in again.
